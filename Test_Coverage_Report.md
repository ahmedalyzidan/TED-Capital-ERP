## 🚀 Getting Started

### Prerequisites
- Node.js (v18+)
- Local development server running at `http://localhost:5173`

### Installation
1. Navigate to the testing directory:
   ```bash
   cd playwright-e2e-tests
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Install Playwright browsers:
   ```bash
   npx playwright install
   ```

### Running Tests
- **Run all tests:**
  ```bash
  npx playwright test
  ```
- **Run with UI mode (Visual Debugging):**
  ```bash
  npx playwright test --ui
  ```
- **Run a specific module:**
  ```bash
  npx playwright test tests/finance.spec.js
  ```

## 🛠️ Implementation Details
- **Architecture**: Page Object Model (POM) ready with separate utilities.
- **Environment Isolation**: Entire suite is contained in `playwright-e2e-tests/` to prevent source code pollution.
- **Cross-Browser Support**: Configured for Chromium, Firefox, and Webkit (Safari engine).
- **Safety**: Uses `@playwright/test` for parallel execution and automatic retries on failure.

## ⚠️ Complex Areas & Manual Review
The following areas use complex interactions that may require manual verification in edge cases:
1. **Dynamic Charts**: While we verify the presence of the `canvas`, verifying exact data points in Chart.js requires visual regression testing (Snapshot comparison).
2. **GPS Attendance**: Geolocation mocking is handled by browser permissions, but real-world GPS accuracy is better tested on physical devices.
3. **Backup Restore**: Tests verify that the "Restore" modal opens, but triggering a full restore in a production-like environment should be done with extreme caution.
