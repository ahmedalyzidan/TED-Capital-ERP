## CORE BUSINESS RULES
1. Ledger entries must balance (debit = credit) — enforced by DB trigger check_ledger_balance()
2. Approval limits table controls who can approve which transaction amount per module
3. JWT access token expires → use refresh token endpoint to renew
4. All financial mutations must post to ledger with a reference_no
5. inventory_items.remaining_qty updated on each sale/purchase
6. Soft-delete pattern: is_deleted=true + deleted_at timestamp (ar_invoices, etc.)
